# ph-mus

`ph-mus` is the public namespace for the Predictive History Museum.

It is the chapter exhibit layer for both conceptual volumes: Volume I / `ph-civ` and Volume II / `ph-apo`. It is not a third volume.

It contains exhibit manifests, artifact metadata, schemas, validation rules, and generated public museum pages. It does not store the full artifact archive. Accepted artifacts must also exist in the local museum vault and in the shared document cloud workspace.

The current public manifest index is `data/museum/index.json`. Use `ph-mus list` and `ph-mus route <source_id>` to inspect available exhibit routes without exposing artifact binaries.
