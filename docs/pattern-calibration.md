# Pattern Calibration

Patterns are public, bridge-ready reference cards. They make recurring civilizational frames easy to cite without turning `ph-civ` into the live strategy workspace.

## Pattern Rules

Each pattern must have:

- `pattern_id`
- `title`
- `summary`
- `strategy_use`
- `source_ids`
- `tags`
- `limits`
- `public_status`

`source_ids` must point to existing public cards and chapter bodies. Pattern records should summarize and cite; they must not duplicate long transcript passages, private notes, strategy-codex paths, hidden bibliography claims, or live current-event assertions.

## Status Values

- `seed`: initial public bridge pattern, useful but not exhaustive.
- `in_review`: actively being revised or expanded.
- `stable`: reviewed enough to be treated as a durable public reference.

## Authoring Posture

Write patterns as careful orientation aids. A pattern should help a reader ask better questions; it should not decide a current case by analogy. Every live application still needs its own evidence.
